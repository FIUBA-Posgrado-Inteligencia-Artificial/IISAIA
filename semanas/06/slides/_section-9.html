Claude Code en el día a día.

/btw <pregunta>

Una pregunta al pasar que no entra al historial ni infla el contexto.

Para chequear un detalle sin ensuciar la sesión en la que estás trabajando.

/rc alias de /remote-control

Deja la sesión disponible para controlarla desde claude.ai.

Para seguir desde el celular o el browser, lejos de la terminal.

claude -c  /  claude -r

Retomás una sesión anterior. -c la última del directorio; -r abre un picker para elegir cuál.

Para no perder el contexto de un día para el otro.

/rewind o Esc Esc

Volvés atrás la conversación y el código a un checkpoint anterior.

Cuando una serie de cambios salió mal y querés deshacer sin pelear con git.

Y una de yapa: /compact resume la conversación cuando el contexto se llena, sin perder el hilo.

Cinco reveals — los cuatro tips de a uno, después la yapa de /compact.

Los cuatro comandos/flujos. Verificados contra la doc de Claude Code. Cada uno con qué hace y cuándo usarlo. El quinto, /compact, como bonus.

"Primero, barra btw, de 'by the way'. Le hacés una pregunta al pasar y la respuesta no entra al historial ni te infla el contexto — aparece y se va. Ideal para chequear un detalle sin ensuciar la sesión en la que estás laburando."

"Segundo, barra rc, que es alias de barra remote-control. Deja la sesión disponible para controlarla desde claude.ai. Arrancás algo en la terminal de la oficina y lo seguís desde el celular, o desde el browser. La sesión es la misma."

"Tercero, retomar sesiones. claude menos c retoma la última sesión de ese directorio, directo. claude menos r abre un picker para que elijas cuál de tus sesiones anteriores retomar. Es la forma de no perder el contexto de un día para el otro — no arrancás de cero cada vez."

"Cuarto, y para mí el más subestimado: barra rewind, o apretar Escape dos veces. Vuelve atrás no solo la conversación sino también el código, a un checkpoint anterior. Cada prompt crea un checkpoint automático. Cuando una serie de cambios se fue al descenso, en vez de pelearte con git para deshacer, hacés rewind y volvés a un punto bueno."

"Y una de yapa para cuando una sesión se hace larga: barra compact resume toda la conversación en algo breve, libera contexto, y seguís sin perder el hilo. Esas cinco, incorpórenlas y van a laburar bastante más cómodos."

El agente ya no está limitado a lo que trae de fábrica.

MCP es cómo lo extendés — y la pregunta cero, cómo elegís bien qué sumar.

Cierre temático de la semana, en una sola idea (no un recap con bullets). Conecta el arco entero: el problema de §1 (el set finito) se resuelve con MCP, y la pregunta cero es la herramienta de criterio. Después viene la slide de preguntas.

"Y con esto cerramos. Arrancamos la clase con un problema: el agente trae un set finito de tools, curado, que no podés extender editando el host. Lo terminamos con la respuesta: MCP es la forma estándar de extender lo que el agente ve y lo que toca, sin atarte a un host. Y si se llevan una sola herramienta de criterio, que sea la pregunta cero: antes de sumar cualquier server, ¿el host ya lo hace? Eso es todo. ¿Preguntas?"
