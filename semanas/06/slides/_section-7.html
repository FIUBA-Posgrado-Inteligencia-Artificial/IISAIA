"Primer demo. Apliquemos la pregunta cero: ¿Claude Code maneja un browser? No — sabe correr Bash, pero no renderiza páginas, no clickea, no completa forms. La respuesta es no, así que corresponde un MCP."

"El server que usamos es Playwright MCP: expone un browser controlado como tools. Y fíjense en una cosa que vamos a contrastar con el próximo demo: este es un server local. Corre en tu propia máquina, comunicándose por Stdio, sin red. Vamos a conectarlo a un proyecto y verlo actuar."

Una línea, y queda.

El comando que ya vimos, aplicado.

$ claude mcp add playwright npx -y @playwright/mcp
local · Stdio El server corre como subproceso en tu máquina. npx baja el paquete la primera vez; después arranca solo en cada sesión.
sin red Los mensajes JSON-RPC viajan por stdin/stdout. No hay URL, no hay auth: el server es local.

Install del server local. Lo importante para el contraste con §8: acá hay un comando que arranca un proceso local (Stdio, sin red, sin auth). En el remoto va a haber una URL y OAuth.

"Se agrega con el comando que ya vimos en el bloque del ecosistema: claude mcp add playwright, npx menos y arroba playwright barra mcp. Una línea."

"Y la forma de este server es la de un local, transport Stdio: corre como un subproceso en tu propia máquina. npx baja el paquete la primera vez y después lo arranca solo cada vez que abrís una sesión. Los mensajes viajan por stdin y stdout, sin red. No hay URL ni autenticación, porque el server vive con vos. Guarden esa imagen — el próximo demo es exactamente lo contrario."
