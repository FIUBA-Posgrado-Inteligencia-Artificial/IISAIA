Un reveal — spaghetti se va, hub aparece.

La matriz formal, abstracta. §1.4 ya mostró el caso concreto (Postgres × runtimes); §2.1 generaliza: N clientes × M proveedores. La transición de spaghetti a hub es la promesa estructural del protocolo. Este diagrama se va a reusar tal cual en §6, cuando hablemos del ecosistema.

"Pongamos en abstracto lo que vimos en concreto. Tenés N clientes — pueden ser agentes, IDEs, runtimes — y tenés M proveedores de capacidades — Postgres, browser, Slack, lo que sea. Si cada cliente define su propia forma de integrar capacidades, entonces cada par cliente-proveedor es una integración custom. Eso es lo que ven a la izquierda: N por M. Cuatro por cuatro, dieciséis líneas. Cliente nuevo, M integraciones más. Proveedor nuevo, N integraciones más. Crece feo, sin techo."

"Si en cambio metés un protocolo común al medio — esto es lo que aparece ahora — la matriz se colapsa. Cada cliente habla el protocolo, una vez. Cada proveedor habla el protocolo, una vez. Total: N más M. Cuatro más cuatro, ocho conexiones. Y la propiedad clave: cliente nuevo, una integración, hereda todos los proveedores. Proveedor nuevo, una integración, queda disponible para todos los clientes. Esto es lo que hace un protocolo. Y este patrón no es nuevo — ya pasó antes."

Este patrón ya pasó antes.

Dos precedentes históricos del mismo movimiento.

1991

HTTP

HyperText Transfer Protocol

browsers × servers

Antes: cada browser hablaba con cada server por su cuenta. Después de HTTP, cualquier browser habla con cualquier server, sin coordinación entre autores. Internet se vuelve posible.

2016

LSP

Language Server Protocol

IDEs × lenguajes

Antes: cada IDE × cada lenguaje, autocompletado a mano. Después de LSP, cada lenguaje publica un server, cada IDE lo consume. La matriz colapsa.

MCP es el mismo movimiento, para agentes × capacidades. No inventa nada — aplica una idea conocida a un dominio nuevo.

Dos reveals — primero LSP, después la línea de cierre con MCP.

Dos precedentes históricos del mismo patrón, revelados de a uno. Ubican a MCP en una tradición — no es invento de Anthropic, es aplicar a agentes una idea que ya resolvió problemas análogos.

"Este movimiento que acabamos de ver — la matriz que se colapsa por un protocolo común — no es nuevo. Ya pasó dos veces en la historia reciente del software, y vale la pena ubicarlo donde lo vieron antes. Primero, HTTP. Año 91. HyperText Transfer Protocol. Antes de HTTP cada browser hablaba con cada servidor por su cuenta — coordinaban formatos, encodings, mecanismos. Después de HTTP, cualquier browser habla con cualquier server. Internet se vuelve posible. Esa es la propiedad del protocolo común: descomprime una matriz de coordinación humana en infraestructura compartida."

"Segundo, LSP — Language Server Protocol. 2016, Microsoft. Resolvió exactamente el mismo problema en el mundo de los IDEs. Antes, cada IDE — VS Code, Vim, Emacs, Sublime — implementaba autocompletado y go-to-definition para cada lenguaje — Python, Go, Rust, TypeScript — a mano. N por M. Insostenible. Después de LSP, cada lenguaje publica un language server, cada IDE consume cualquier server. N más M. Y eso es lo que hace que hoy cualquier IDE moderno tenga autocompletado decente en cualquier lenguaje, sin que el equipo del IDE escriba código específico para cada uno."

"MCP es el mismo movimiento, aplicado a agentes y capacidades. No inventa nada conceptualmente. Aplica una idea conocida — protocolo común — a un dominio donde el problema apareció hace poco."

la definición oficial

MCP es un estándar open-source para conectar aplicaciones de IA a sistemas externos.

— modelcontextprotocol.io

Lo propuso Anthropic en 2024. Lo adoptaron:

Claude ChatGPT Cursor Codex CLI Opencode VS Code

La definición oficial. Nota deliberada: la definición no menciona a Anthropic. Es un estándar abierto que Anthropic propuso, no un producto Anthropic.

"Esta es la definición oficial, de la doc de modelcontextprotocol.io. Léanla con cuidado: 'MCP es un estándar open-source para conectar aplicaciones de IA a sistemas externos.' Notá lo que esa frase no dice. No dice 'estándar de Anthropic'. No dice 'protocolo de Claude'. Dice 'estándar open-source'. Es deliberado: aunque lo propuso Anthropic en 2024, el protocolo es abierto y existe independientemente."

"La adopción lo confirma. Claude lo soporta, obviamente. Pero también ChatGPT, Cursor, Codex CLI, Opencode, VS Code, y otros. Cuando ustedes aprenden MCP, aprenden algo que viaja con ustedes a cualquier runtime de IA que vayan a usar. Eso es lo que hace que valga la pena estudiarlo."

la analogía oficial

MCP es como un puerto USB-C para aplicaciones de IA.

Así como USB-C estandariza la forma de conectar dispositivos electrónicos, MCP estandariza la forma en que las aplicaciones de IA se conectan a sistemas externos.

— modelcontextprotocol.io

La analogía oficial, en versión de bolsillo. HTTP y LSP explicaron el porqué estructural; USB-C es la metáfora rápida que se quedan.

"Y la doc oficial propone una analogía más corta, más memorable, que les va a venir a la cabeza después: MCP es como un puerto USB-C para aplicaciones de IA. La idea es la misma — antes de USB-C cada dispositivo tenía su propio cable, su propio conector, su propia forma de comunicarse. Después de USB-C, cualquier dispositivo con USB-C se conecta a cualquier otro con USB-C, sin pensar."

"HTTP y LSP les sirven para entender el porqué estructural — el N por M que colapsa a N más M. USB-C les sirve como mnemónico, para acordarse del concepto en dos palabras. Las dos son útiles."
